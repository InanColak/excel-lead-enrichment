    `map_enrichment_status(row_status: str, contact: Contact | None) -> str`:
    Per D-55:
    - row_status "enriched" AND contact has phone -> "enriched"
    - row_status "enriched" AND contact has no phone -> "email_only"
    - row_status "email_only" -> "email_only"

    `async def generate_output_file(job_id: uuid.UUID, session_factory: async_sessionmaker) -> str`:
    Per D-56/D-57/D-64:
    a. Open session from session_factory
    b. Load Job by job_id
    c. Load all JobRows ordered by row_index, eagerly load contact_id references
    d. Build enrichment_map: dict[int, dict] mapping row_index -> {"email": str|None, "phone": str|None, "status": str}
       - For each row: if row.contact_id, load the Contact (batch query all contact_ids first to avoid N+1)
       - Use map_enrichment_status for status value
    e. Read original file with `load_workbook(job.file_path, read_only=True, data_only=True)`
    f. Create new `Workbook()` for output
    g. Copy header row from original, append 3 headers: "enriched_email", "enriched_phone", "enrichment_status"
    h. For each data row (row_index 0-based maps to openpyxl row 2+):
       - Copy all original cell values
       - Append enrichment data from enrichment_map[row_index]
       - CRITICAL: row_index 0 = original file row 2 (after header). Iterate ws_read rows skipping header, maintain counter starting at 0.
    i. Save to output_path: `Path(job.file_path).parent / f"{Path(job.filename).stem}_enriched.xlsx"`
    j. Update job.output_file_path = str(output_path)
    k. Commit and return str(output_path)

    Close the read_only workbook in a finally block.

| T-04-02 | Information Disclosure | output file content | accept | Output file contains data the job owner uploaded plus enrichment results they requested. No cross-user data leakage since JobRows are scoped by job_id. |
