Create `app/enrichment/service.py` with the following functions. This is the core business logic — no Celery dependency, pure async functions accepting a session factory.

**1. `extract_field(raw_data: dict, column_mappings: list[dict], field_type: str) -> Optional[str]`**
   - Finds the column mapping entry where `detected_type == field_type`
   - Returns `raw_data[column_name]` stripped and lowered (for email/linkedin_url) or stripped (for names/company)
   - Returns None if not found or empty
   - `field_type` values: `"email"`, `"linkedin_url"`, `"first_name"`, `"last_name"`, `"company"`

**2. `build_dedup_groups(rows: list[JobRow], column_mappings: list[dict]) -> dict[str, list[JobRow]]`** (per D-48)
   - Groups rows by normalized contact identity
   - Key logic:
     - If row has email: key = `f"email:{email.strip().lower()}"`
     - Elif row has linkedin_url: key = `f"linkedin:{linkedin_url.strip().lower()}"`
     - Else: key = `f"row:{str(row.id)}"` (treat as unique, will need name+company for Apollo)
   - Returns `defaultdict(list)` mapping key -> list of JobRow objects
   - Rows with status != "pending" are skipped (already processed or errored)

**3. `async def batch_contact_lookup(db: AsyncSession, groups: dict[str, list[JobRow]]) -> dict[str, Contact]`** (per D-47, Pitfall 6)
   - Extracts all email keys and all linkedin keys from groups
   - Single batch query: `SELECT * FROM contacts WHERE email IN (:emails)` for email-keyed groups
   - Second batch query: `SELECT * FROM contacts WHERE linkedin_url IN (:linkedin_urls)` for linkedin-keyed groups
   - Returns dict mapping group key -> Contact for found contacts
   - Uses `select(Contact).where(Contact.email.in_(email_list))` SQLAlchemy syntax

**4. `async def process_job(job_id: uuid.UUID, session_factory: async_sessionmaker) -> None`** (per D-50)
   - Main orchestration function. Steps:

   a. Open a session, load Job by id. Transition status from "confirmed" to "processing". Commit.

   b. Load all JobRow records where `job_id == job_id` and `status == "pending"` ordered by row_index.

   c. Build dedup groups via `build_dedup_groups(rows, job.column_mappings)`.

   d. Batch contact lookup via `batch_contact_lookup(db, groups)`.

   e. Read API key from DB via `_get_api_key_from_db(db)`. Build webhook_url: `f"{settings.webhook_base_url}/api/v1/webhooks/apollo"` if `settings.webhook_base_url` is non-empty, else None.

   f. Create `ApolloClient(api_key=api_key, webhook_url=webhook_url)`.

   g. Initialize counters: `cache_hits = 0`, `api_calls = 0`, `processed = 0`.

   h. For each group in dedup_groups:
      - If group key found in batch_contact_lookup results (cache hit):
        - Link all rows in group to existing contact: set `row.contact_id = contact.id`, `row.status = "enriched"`
        - Increment `cache_hits += 1`, `processed += len(group_rows)`
      - Else (cache miss — call Apollo):
        - Extract identifiers from first row in group using `extract_field()`
        - Try `apollo_client.enrich_person(first_name, last_name, organization_name, email, linkedin_url)`
        - On success:
          - Create or update Contact record: set email (from `response.person.email`), first_name, last_name, company, linkedin_url, `apollo_id = response.person.id`, `raw_apollo_response = response data as dict`
          - Link all rows in group to this contact: `row.contact_id = contact.id`, `row.status = "enriched"`
          - Increment `api_calls += 1`, `processed += len(group_rows)`
        - On ApolloNotFoundError:
          - Mark all rows in group: `row.status = "not_found"`, `row.error_message = "Contact not found in Apollo"`
          - Increment `api_calls += 1`, `processed += len(group_rows)` (per D-40)
        - On ApolloClientError (e.g., bad API key):
          - Mark all rows in group: `row.status = "error"`, `row.error_message = str(error)`
          - Increment `processed += len(group_rows)`
          - Log error but continue processing other groups
        - On ApolloTransientError (after retries exhausted):
          - Mark all rows in group: `row.status = "not_found"`, `row.error_message = f"Apollo API failure after retries: {error}"` (per D-40)
          - Increment `api_calls += 1`, `processed += len(group_rows)`

      - Every 50 rows processed (or at end): flush progress to Job model:
        ```python
        job.processed_rows = processed
        job.cache_hits = cache_hits
        job.api_calls = api_calls
        await db.commit()
        ```

   i. After all groups processed: close apollo_client. Transition job to "awaiting_webhooks" if any api_calls were made (webhooks expected). If api_calls == 0 (all cache hits), transition directly to "complete" or "partial" based on D-52 logic.

   j. Final commit with all metrics.

   k. Return (task.py will handle scheduling the webhook checker).

   **Important implementation details:**
   - Use `try/finally` to ensure `apollo_client.close()` is called
   - Wrap the entire function in try/except for catastrophic errors: on unrecoverable error, set `job.status = "failed"`, commit, re-raise
   - Per Pitfall 1: create session from `session_factory()` directly, NOT from `get_db()` FastAPI dependency
   - Per Pitfall 6: batch lookup, not N+1 queries
   - Per ENRICH-09: never read or modify the original .xlsx file — all data comes from JobRow.raw_data
   - Per ENRICH-08: each job gets its own task invocation with its own DB session — no shared state
   - Per ENRICH-01: rows are identified by UUID throughout (JobRow.id)

**1. Create `app/enrichment/tasks.py`** — Two Celery tasks:

```python
import asyncio
import logging
import uuid

from sqlalchemy import select
from sqlalchemy.ext.asyncio import AsyncSession, async_sessionmaker, create_async_engine

from app.celery_app import celery_app
from app.config import settings
from app.enrichment.service import process_job
from app.jobs.models import Job, JobRow, RowStatus

logger = logging.getLogger(__name__)


def _get_session_factory() -> async_sessionmaker[AsyncSession]:
    """Create a standalone async session factory for Celery tasks.
    Per Pitfall 1: Celery runs outside FastAPI — cannot use get_db().
    Per Pitfall 4: creates engine per task invocation (acceptable for job-level granularity).
    """
    engine = create_async_engine(settings.database_url, echo=False)
    return async_sessionmaker(engine, expire_on_commit=False)


@celery_app.task(bind=True, acks_late=True, reject_on_worker_lost=True, time_limit=3600)
def process_enrichment_job(self, job_id: str):
    """Celery task: process a confirmed enrichment job.
    Per D-50: single orchestrator task per job.
    Per Pitfall 2: job_id passed as str, converted to UUID inside.
    """
    logger.info(f"Starting enrichment job {job_id}")
    session_factory = _get_session_factory()
    try:
        asyncio.run(_run_enrichment(job_id, session_factory))
    except Exception:
        logger.exception(f"Enrichment job {job_id} failed")
        # Mark job as FAILED on catastrophic error
        asyncio.run(_mark_job_failed(job_id, session_factory))
        raise


async def _run_enrichment(job_id: str, session_factory: async_sessionmaker):
    """Run the enrichment pipeline and schedule webhook checker if needed."""
    job_uuid = uuid.UUID(job_id)
    await process_job(job_uuid, session_factory)

    # Check if job is now AWAITING_WEBHOOKS — if so, schedule the timeout checker
    async with session_factory() as db:
        result = await db.execute(select(Job).where(Job.id == job_uuid))
        job = result.scalar_one_or_none()
        if job and job.status == "awaiting_webhooks":
            # Per D-53: schedule delayed check after webhook_timeout_seconds (default 300 = 5 min)
            check_webhook_completion.apply_async(
                args=[job_id],
                countdown=settings.webhook_timeout_seconds,
            )
            logger.info(f"Scheduled webhook completion check for job {job_id} in {settings.webhook_timeout_seconds}s")


async def _mark_job_failed(job_id: str, session_factory: async_sessionmaker):
    """Mark a job as FAILED after catastrophic error."""
    try:
        async with session_factory() as db:
            result = await db.execute(select(Job).where(Job.id == uuid.UUID(job_id)))
            job = result.scalar_one_or_none()
            if job:
                job.status = "failed"
                await db.commit()
    except Exception:
        logger.exception(f"Failed to mark job {job_id} as failed")


@celery_app.task(bind=True, acks_late=True)
def check_webhook_completion(self, job_id: str):
    """Celery task: check if all webhooks arrived after timeout period.
    Per D-53: scheduled with countdown=300 (5 minutes) after API calls complete.
    Per D-52: determines final job status (COMPLETE vs PARTIAL).
    """
    logger.info(f"Checking webhook completion for job {job_id}")
    session_factory = _get_session_factory()
    asyncio.run(_check_webhook_completion_async(job_id, session_factory))


async def _check_webhook_completion_async(job_id: str, session_factory: async_sessionmaker):
    """Check webhook completion and finalize job status.
    Per D-53: mark remaining rows email-only when phone webhook timed out.
    """
    job_uuid = uuid.UUID(job_id)
    async with session_factory() as db:
        result = await db.execute(select(Job).where(Job.id == job_uuid))
        job = result.scalar_one_or_none()
        if not job or job.status != "awaiting_webhooks":
            logger.info(f"Job {job_id} is not awaiting webhooks (status={job.status if job else 'missing'}), skipping")
            return

        # Count rows that were enriched but whose contact still has no phone
        # These are the ones waiting for webhooks
        result = await db.execute(
            select(JobRow)
            .where(JobRow.job_id == job_uuid)
            .where(JobRow.status == RowStatus.ENRICHED.value)
        )
        enriched_rows = result.scalars().all()

        # Load contacts for enriched rows to check phone status
        from app.contacts.models import Contact
        webhook_timeouts = 0
        for row in enriched_rows:
            if row.contact_id:
                contact_result = await db.execute(select(Contact).where(Contact.id == row.contact_id))
                contact = contact_result.scalar_one_or_none()
                if contact and contact.apollo_id and not contact.phone:
                    # Per D-53: This contact had an Apollo API call (has apollo_id) but no phone
                    # arrived via webhook. Mark the JobRow as EMAIL_ONLY so Phase 4 Excel
                    # generation can distinguish these rows without fragile contact field inference.
                    row.status = "email_only"
                    webhook_timeouts += 1
                    logger.info(f"Row {row.id} marked email_only: webhook timed out for apollo_id={contact.apollo_id}")

        job.webhook_timeouts = webhook_timeouts

        # Determine final status per D-52:
        # Re-count row statuses after EMAIL_ONLY updates
        all_rows_result = await db.execute(
            select(JobRow).where(JobRow.job_id == job_uuid)
        )
        all_rows = all_rows_result.scalars().all()

        enriched_count = sum(1 for r in all_rows if r.status in (RowStatus.ENRICHED.value, "email_only"))
        not_found_count = sum(1 for r in all_rows if r.status == RowStatus.NOT_FOUND.value)
        error_count = sum(1 for r in all_rows if r.status in (RowStatus.ERROR.value, RowStatus.SKIPPED.value))

        if enriched_count == 0 and (not_found_count > 0 or error_count > 0):
            job.status = "failed"
        elif not_found_count > 0 or error_count > 0:
            job.status = "partial"
        else:
            job.status = "complete"

        await db.commit()
        logger.info(f"Job {job_id} finalized: status={job.status}, webhook_timeouts={webhook_timeouts}")
```

**Key implementation details:**
- Per Pitfall 2: `job_id` is `str`, converted to `uuid.UUID()` inside async functions
- Per Pitfall 1: `_get_session_factory()` creates standalone engine, no FastAPI deps
- `acks_late=True` and `reject_on_worker_lost=True` per Celery best practices for data-integrity tasks
- `time_limit=3600` (1 hour) prevents runaway tasks for very large jobs
- `check_webhook_completion` uses D-52 logic for final status determination
- **Per D-53:** `_check_webhook_completion_async` now updates individual JobRow records to `"email_only"` status for contacts where `apollo_id` is set but `phone` is empty (webhook timed out). This allows Phase 4 Excel generation to distinguish "email-only timeout" rows from "fully enriched" rows without fragile contact field inference.
