# Requirements Archive — v1.0

**Shipped:** 2026-04-07
**Core Value:** Every uploaded Excel file comes back with accurate email addresses and phone numbers — without wasting Apollo API credits on data we already have.

## v1 Requirements (All Complete)

### File Processing
- [x] **FILE-01**: User can upload Excel files (.xlsx/.xls) via the REST API — Phase 2
- [x] **FILE-02**: System validates uploaded file on receipt (format, size limit, sheet structure) — Phase 2
- [x] **FILE-03**: System auto-detects column types from headers and content sampling — Phase 2
- [x] **FILE-04**: User can review and override auto-detected column mappings via API — Phase 2
- [x] **FILE-05**: System handles bad/malformed rows gracefully without failing the entire job — Phase 2

### Enrichment Pipeline
- [x] **ENRICH-01**: System assigns a unique UUID to each row at parse time — Phase 2 (early delivery)
- [x] **ENRICH-02**: System deduplicates contacts within a single upload — Phase 3
- [x] **ENRICH-03**: System checks local contact database first before any Apollo API call — Phase 3
- [x] **ENRICH-04**: System calls Apollo People Enrichment API; email in immediate response, phone via webhook — Phase 3
- [x] **ENRICH-05**: System stores all Apollo enrichment results in local contact database — Phase 3
- [x] **ENRICH-06**: System marks rows where Apollo returns no match with "not found" status — Phase 3
- [x] **ENRICH-07**: System processes large files (1,000+ rows) as background jobs — Phase 3
- [x] **ENRICH-08**: System isolates concurrent jobs for multiple users — Phase 3
- [x] **ENRICH-09**: System preserves the original uploaded file — Phase 2
- [x] **ENRICH-10**: System tracks per-job metrics (cache hits, API calls, webhook stats) — Phase 3
- [x] **ENRICH-11**: System exposes webhook receiver for Apollo phone-data callbacks with auth — Phase 3

### Job Lifecycle
- [x] **JOB-01**: System assigns unique job ID tracking full lifecycle — Phase 3

### Output & History
- [x] **OUTPUT-01**: User can download enriched Excel with original + appended columns — Phase 4
- [x] **OUTPUT-02**: User can query real-time job status and progress — Phase 4
- [x] **OUTPUT-03**: User can list job history and re-download past results — Phase 4

### Authentication & Admin
- [x] **AUTH-01**: User can log in with email and password — Phase 1
- [x] **AUTH-02**: Admin can configure shared Apollo API key — Phase 1
- [x] **AUTH-03**: Admin can add and remove team members — Phase 1
- [x] **AUTH-04**: User can query usage stats — Phase 4

### Infrastructure
